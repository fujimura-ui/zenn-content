## 「東京都の製造業で使える補助金、締切順に教えて」が一発で返ってくる

Claude Desktop にMCPサーバーを1本繋ぐと、こういう会話ができるようになります。

> **私**：東京都の会社で使える設備投資系の補助金、締切が近い順に教えて
>
> **Claude**：現在募集中のものは11件あります。直近では——
> ・事業承継・M&A補助金（15次）… 上限1,000万円、締切 7/24（残り15日）
> ・ものづくり・商業・サービス生産性向上促進補助金（19次）… 上限4,000万円、締切 9/28
> （各件に公式の申請ページURL付き）

裏で動いているのはスクレイピングではなく、**デジタル庁のjGrants公式API**です。国のデータベースをAIが直接引いているので、情報は常に最新で、出典も明確です。

この調子で、日本の公的データをAIエージェントから使えるようにするMCPサーバーを**9本**作りました。この記事はその紹介と、作ってみて分かったことのまとめです。

## なぜ作ったか：日本の政府APIは「宝の山なのに誰も繋いでいない」

日本の官公庁は、実はかなりの数の公式APIを無料公開しています。補助金（jGrants）、法令全文（e-Gov）、国会会議録（国立国会図書館）、企業情報100万社超（gBizINFO）、気象・地震（気象庁）、有価証券報告書（EDINET）、政府統計（e-Stat）、不動産の実際の成約価格（国土交通省）——。

ところが2026年7月時点で、これらをMCP（Model Context Protocol：AIが道具を使うための共通規格）で使えるようにしたものは、探した限りほぼ存在しませんでした。英語圏ではMCPサーバーが2万本以上流通しているのに、日本の公的データはぽっかり空白だったのです。

無いなら作ればいい、ということで9本作りました。

## 作った9本

| # | 名前 | できること | データ源 |

| 1 | Japan Subsidy Search | 募集中の補助金・助成金を検索（締切・上限額・申請URL付き） | jGrants（デジタル庁） |
| 2 | Japan Law Search | 現行法令 約1万本の全文検索・条文取得 | e-Gov法令API |

| 4 | Japan Company Leads Finder | 企業100万社超を規模・地域・補助金受給歴で検索 | gBizINFO（経産省） |
| 5 | Japan Weather & Alerts | 天気予報・気象警報・地震情報 | 気象庁 |
| 6 | Japan Address & Geocoding | 住所正規化・郵便番号・緯度経度変換 | 国土地理院ほか |
| 7 | Japan Corporate Filings | 有価証券報告書・大量保有報告の検索 | EDINET（金融庁） |

すべて**リモートMCP**（URLを設定に足すだけ、インストール不要）です。

## 試し方（30秒）

Claude Desktop の場合：「設定 → コネクタ → カスタムコネクタを追加」で、たとえば補助金検索なら次のURLを追加するだけです。

```
https://hojokin-mcp.plus-synergy.workers.dev/mcp
```

Claude Code なら1コマンド：

```bash
claude mcp add --transport http japan-subsidy https://hojokin-mcp.plus-synergy.workers.dev/mcp
```

他の8本も同じ形式（`horei` / `kokkai` / `gbiz-leads` / `tenki` / `jusho` / `edinet` / `estat` / `fudousan` + `-mcp.plus-synergy.workers.dev/mcp`）で公開しています。無料エンドポイントはIPごと1分10コールのレート制限があります。本格利用・商用利用向けには [Apify Store 版](https://apify.com/e-asakura)（従量課金 $0.02/コール）を用意しています。

## 実際に何ができるか：3つの実演

### ① 補助金の「見落とし」を拾う

国が公募中の補助金は今日時点で311件。うち締切14日以内が45件あります。人間がjGrantsを毎日見に行くのは無理ですが、エージェントなら「毎朝、自社が対象の補助金で締切が近いものを教えて」が実現できます。

### ② 法令の条文をピンポイントで引く

「下請代金支払遅延等防止法の第4条の原文を出して」→ e-Gov APIから正確な条文が返ります。約1万本の現行法令が対象なので、契約書チェックAIの参照元や、社内規程づくりの根拠確認に使えます。

### ③ 国会から「次の政策」を先読みする

国会会議録APIは意外な掘り出し物でした。たとえば「蓄電池 支援」で2026年の発言を検索すると26件ヒットし、環境省審議官の「避難施設への蓄電池導入補助を行っている」といった答弁が原文リンク付きで取れます。国会での議論は将来の補助金・規制の"予告編"なので、業界動向のモニタリングに向いています。

## 技術構成

- **ホスティング**：Cloudflare Workers + Durable Objects（`agents` パッケージの `McpAgent`）。リモートMCPのStreamable HTTP / SSE両対応
- **実装**：TypeScript + MCP公式SDK。コアロジックは stdio 版と Workers 版で共有
- **課金版**：Apify Store の Pay Per Event（ツールコール単位で課金）
- **原価**：データ源はすべて無料の公式APIなので、インフラ費のみ

構成の詳細は各リポジトリ（[GitHub: fujimura-ui](https://github.com/fujimura-ui)）で公開しています。

## ハマった話：公開5日目、bot に無料枠を食い破られる

公開してMCPディレクトリ（Glama・公式レジストリ・mcp.so等）に登録した途端、**世界中のディレクトリの死活監視botが数分おきにアクセスしてくる**ようになりました。

初期実装ではレート制限をDurable Objectsのストレージで数えていたため、リクエストのたびにDB書き込みが発生。Cloudflare無料枠（10万書き込み/日）が毎日溶け、ある朝9本全部が `Error 1102` で停止していました。

対策は2つ：レート制限をメモリ内カウンターに書き換えて書き込みをゼロにしたこと、Workers Paidプラン（$5/月）に加入したこと。**「ディレクトリに載る＝監視botに愛される」**は、リモートMCPを公開する人は知っておいて損がないはずです。
